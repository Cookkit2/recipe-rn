# Lean PR CI Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make PR #389 CI high-signal by simplifying pull request checks, moving heavy security scans off the PR path, and fixing the remaining actionable Jest failures.

**Architecture:** `ci.yml` becomes the only pull-request gate for Bun install, typecheck, formatting, Jest, and one high/critical dependency audit. `security-scan.yml` keeps CodeQL, Semgrep, OSSF Scorecard, and repository policy checks for scheduled/manual/protected-branch runs. Test fixes stay scoped to the failing suites so workflow cleanup does not mask real regressions.

**Tech Stack:** GitHub Actions, Bun, Expo/React Native, Jest/ts-jest, WatermelonDB, TypeScript.

---

## File Structure

- Modify `.github/workflows/ci.yml`: simplify PR CI to quality, Jest, and one Bun audit; remove duplicate build and coverage gates from PRs.
- Modify `.github/workflows/security-scan.yml`: remove PR trigger, remove duplicate Bun audit job, and keep security/policy scans on scheduled/manual/main/master runs.
- Modify `utils/__tests__/gemini-api.test.ts`: update pricing expectation for `gemini-2.5-flash-lite`.
- Modify `data/db/repositories/__tests__/TailoredRecipeMappingRepository.test.ts`: update `database.batch` expectations to array-form calls.
- Investigate and then modify either `data/api/pantryApi.ts` or `data/api/__tests__/pantryApi-addPantryItemsWithMetadata.test.ts`: resolve whether duplicate stock quantity should aggregate from 4 to 6 or remain 4.
- Modify `jest.config.js` only if the legacy `src/` test suites should remain in the active suite; otherwise update test ignore patterns to quarantine legacy `src/` tests explicitly.
- Update `.cursor/scratchpad.md`: record each execution checkpoint and verification result.

Commits are intentionally omitted from task steps because this environment only commits when the user explicitly asks.

Use this workflow content:

```yaml
name: CI

on:
  push:
    branches: [main, master, develop, staging]
  pull_request:
    branches: [main, master, develop, staging]

env:
  NODE_VERSION: "20"

jobs:
  quality:
    runs-on: ubuntu-latest
    name: Quality
    steps:
      - uses: actions/checkout@v6

      - name: Setup Bun
        uses: oven-sh/setup-bun@v2

      - name: Setup Node.js
        uses: actions/setup-node@v6
        with:
          node-version: ${{ env.NODE_VERSION }}

      - name: Cache Bun dependencies
        uses: actions/cache@v4
        with:
          path: ~/.bun/install/cache
          key: ${{ runner.os }}-bun-${{ hashFiles('bun.lockb', 'bun.lock') }}
          restore-keys: |
            ${{ runner.os }}-bun-

      - name: Install dependencies
        run: bun install --frozen-lockfile

      - name: TypeScript check
        run: bun run typecheck

      - name: Prettier check
        run: bunx prettier --check .

  test:
    runs-on: ubuntu-latest
    name: Jest
    needs: quality
    steps:
      - uses: actions/checkout@v6

      - name: Setup Bun
        uses: oven-sh/setup-bun@v2

      - name: Setup Node.js
        uses: actions/setup-node@v6
        with:
          node-version: ${{ env.NODE_VERSION }}

      - name: Cache Bun dependencies
        uses: actions/cache@v4
        with:
          path: ~/.bun/install/cache
          key: ${{ runner.os }}-bun-${{ hashFiles('bun.lockb', 'bun.lock') }}
          restore-keys: |
            ${{ runner.os }}-bun-

      - name: Install dependencies
        run: bun install --frozen-lockfile

      - name: Run Jest
        run: bun run test

  dependency-audit:
    runs-on: ubuntu-latest
    name: Dependency Audit
    steps:
      - uses: actions/checkout@v6

      - name: Setup Bun
        uses: oven-sh/setup-bun@v2

      - name: Setup Node.js
        uses: actions/setup-node@v6
        with:
          node-version: ${{ env.NODE_VERSION }}

      - name: Cache Bun dependencies
        uses: actions/cache@v4
        with:
          path: ~/.bun/install/cache
          key: ${{ runner.os }}-bun-${{ hashFiles('bun.lockb', 'bun.lock') }}
          restore-keys: |
            ${{ runner.os }}-bun-

      - name: Install dependencies
        run: bun install --frozen-lockfile

      - name: Run Bun audit
        run: bun audit --audit-level=high
```

## Task 2: Move Heavy Security Scans Off Pull Requests

**Files:**
- Modify: `.github/workflows/security-scan.yml`

- [ ] **Step 1: Replace the security workflow with scheduled/protected-branch scans**

Use this workflow content:

```yaml
name: Security Scanning

on:
  push:
    branches: [main, master]
  schedule:
    - cron: "0 2 * * *"
  workflow_dispatch:

jobs:
  codeql:
    name: CodeQL SAST
    runs-on: ubuntu-latest
    permissions:
      actions: read
      contents: read
      security-events: write
    steps:
      - uses: actions/checkout@v6
        with:
          fetch-depth: 0

      - name: Initialize CodeQL
        uses: github/codeql-action/init@v4
        with:
          languages: javascript, typescript
          queries: security-extended, security-and-quality

      - name: Autobuild
        uses: github/codeql-action/autobuild@v4

      - name: Perform CodeQL Analysis
        uses: github/codeql-action/analyze@v4
        with:
          category: "/language:javascript-typescript"

  semgrep:
    name: Semgrep SAST
    runs-on: ubuntu-latest
    container:
      image: returntocorp/semgrep
    steps:
      - uses: actions/checkout@v6

      - name: Run Semgrep
        run: |
          semgrep scan \
            --config auto \
            --severity ERROR \
            --sarif \
            --output semgrep-report.sarif \
            || true

      - name: Upload Semgrep results
        uses: github/codeql-action/upload-sarif@v4
        if: always()
        with:
          sarif_file: semgrep-report.sarif

      - name: Check for critical vulnerabilities
        run: |
          if [ -f semgrep-report.sarif ]; then
            CRITICAL=$(jq '[.runs[].results[]? | select(.level == "error")] | length' semgrep-report.sarif)
            if [ "$CRITICAL" -gt 0 ]; then
              echo "Found $CRITICAL critical Semgrep issues"
              exit 1
            fi
          fi
          echo "No critical Semgrep issues found"

  security-policy:
    name: Security Policy Compliance
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v6

      - name: Check for SECURITY.md
        run: |
          if [ ! -f "SECURITY.md" ]; then
            echo "SECURITY.md file not found"
            exit 1
          fi
          echo "SECURITY.md file exists"

      - name: Check for sensitive file exposure
        run: |
          echo "Checking for potentially exposed sensitive files..."
          SENSITIVE_FILES=$(git ls-files | grep -E '(^|/)(\.env|.*\.(key|pem|p12|pfx))$' | grep -v '^ios/\.xcode\.env$' || true)
          if [ -n "$SENSITIVE_FILES" ]; then
            echo "$SENSITIVE_FILES"
            echo "Sensitive files found in git"
            exit 1
          fi
          echo "No sensitive files exposed"

  scorecard:
    name: OSSF Scorecard
    runs-on: ubuntu-latest
    permissions:
      contents: read
      security-events: write
      id-token: write
    steps:
      - uses: actions/checkout@v6
        with:
          persist-credentials: false

      - name: Run Scorecard
        uses: ossf/scorecard-action@v2.3.1
        with:
          results_file: results.sarif
          results_format: sarif
          publish_results: true

      - name: Upload Scorecard results
        uses: github/codeql-action/upload-sarif@v4
        with:
          sarif_file: results.sarif

  security-summary:
    runs-on: ubuntu-latest
    name: Security Scan Summary
    needs: [codeql, semgrep, security-policy, scorecard]
    if: always()
    steps:
      - name: Check security jobs
        run: |
          FAILED=0
          echo "CodeQL: ${{ needs.codeql.result }}"
          echo "Semgrep: ${{ needs.semgrep.result }}"
          echo "Security Policy: ${{ needs.security-policy.result }}"
          echo "Scorecard: ${{ needs.scorecard.result }}"

          if [ "${{ needs.codeql.result }}" != "success" ]; then FAILED=1; fi
          if [ "${{ needs.semgrep.result }}" != "success" ]; then FAILED=1; fi
          if [ "${{ needs.security-policy.result }}" != "success" ]; then FAILED=1; fi
          if [ "${{ needs.scorecard.result }}" != "success" ]; then FAILED=1; fi

          if [ $FAILED -eq 1 ]; then
            echo "Security scanning failed"
            exit 1
          fi
          echo "All security scans passed"
```
